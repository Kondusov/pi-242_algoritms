Задача: Оптимизация системы хранения и поиска ресурсов
Контекст:
Вы разрабатываете систему управления ресурсами для игры/приложения.
У вас есть два типа данных:

Массив предметов с их ценностями (целые положительные числа)

Массив строк-идентификаторов для категорий предметов

Часть 1: Быстрый поиск ресурсов по ценности 
Перед поиском предметов необходимо отсортировать их по возрастанию ценности для возможности применения бинарного поиска.

Сортировка: Реализуйте пузырьковую сортировку для массива цен предметов.

Бинарный поиск: Реализуйте функцию бинарного поиска, которая находит предмет с заданной ценностью в отсортированном массиве.

Особенность: Если предметов с искомой ценностью несколько, функция должна вернуть индекс первого найденного (крайний левый).

Пример:

text
Исходный массив: [15, 3, 8, 15, 20, 7, 15]
После сортировки: [3, 7, 8, 15, 15, 15, 20]
Поиск значения 15 → индекс 3 (первое вхождение)
Поиск значения 25 → -1 (не найдено)  

Задача2

Оптимальное распределение средств (используйте жадный алгоритм)
Есть несколько счетов с дефицитом и профицитом средств, и общий резерв.
Реализуйте жадный алгоритм для оптимального распределения:
50 - это значение при котором счет уже не испытывает дефицит.
более 100 - это излишне для элемента (профицит)
Соберите излишки с элементов превыщающие максимальное значение (100) и положите излишки в резерв.
Распределите средства из резерва сначала на счета испытывающие дефицит (менее 50)
